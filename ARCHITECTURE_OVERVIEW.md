# Project Architecture Overview

This project uses a **Brownfield Shared SDK Architecture**. This means the feature logic is built once (in React Native) and then packaged as a native Android SDK to be used by any host app.

## Project Structure Explained

### 1. `circles-sg-rn-expo` (The Source Repo)
- **Role**: This is the **Shared Feature Repository**.
- **What's inside**: The React Native UI, business logic, and Expo configurations for the "Circles Roaming" feature.
- **Why the `android/` folder exists**: This is the **Development Shell**. It's automatically generated by Expo to allow you to run and test the React Native code locally on an Android device/emulator. 
- **Usage**: You work here when you want to change the UI or logic of the Roaming feature.

### 2. `sg-circles-android-sdk` (The Delivery Repo)
- **Role**: This is the **Distribution Repository**.
- **What's inside**: A dedicated native Android Library project (`circles-travel-pass-sdk`) that packages the React Native feature into a versioned **AAR (Android Archive)**.
- **Why it's needed**: It converts your React Native code into a format that a native Android developer can use just like any other library (like Retrofit or Glide). 
- **Usage**: This repo handles the "packaging and shipping" of the feature.

### 3. `sg-circles-android-host` (The Consumer Repo)
- **Role**: This is the **Native Host App**.
- **What's inside**: A pure native Android application (no React Native code directly).
- **Consumption**: It adds the SDK AAR from Step 2 as a dependency and "launches" the roaming feature via a simple Intent.

---

## The Big Picture: Why two Android projects?

| Component | Purpose | Key File |
| :--- | :--- | :--- |
| **`circles-sg-rn-expo/android`** | **Testing the Feature**: A shell to run the React Native code during development. | `MainApplication.kt` |
| **`sg-circles-android-sdk/...`** | **Packaging the SDK**: A library project to bundle the feature for distribution. | `build.gradle` (AAR config) |

**In short: You *develop* the feature in the first one, and you *publish* it to a native library in the second one.**
